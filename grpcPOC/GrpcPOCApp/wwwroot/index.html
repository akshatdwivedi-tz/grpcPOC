gRPC POC — Chat Demo

Demonstrating all four gRPC communication patterns via a REST bridge

Mode 1 — Unary RPC

Client sends one request and waits for one response.
rpc SendMessage (MessageRequest) returns (MessageResponse)
REST bridge: POST /api/chat/send

Ready — fill in the form and click Send.

Each click makes a single round-trip: request → server stores message → response with message ID.

Mode 2 — Server Streaming RPC

Client sends one request; the server streams multiple messages back one-by-one.
rpc GetChatHistory (HistoryRequest) returns (stream MessageRequest)
REST bridge: GET /api/chat/history (SSE)

Ready — click "Fetch History" to watch messages stream in one-by-one.

Messages arrive with a small delay between each, just like the gRPC server-streaming mode sends them one at a time over a single HTTP/2 connection.

Mode 3 — Client Streaming RPC

Client streams multiple messages; server sends one summary response when done.
rpc BulkUploadMessages (stream MessageRequest) returns (UploadResponse)
REST bridge: POST /api/chat/bulk-upload

Queue is empty — add messages above.
Ready — build a queue then click "Upload All" to send all messages in one streaming call.

All queued messages are sent as a single "stream" to the server, which returns one aggregate response after processing every message.

Mode 4 — Bidirectional Streaming RPC

Client and server both stream simultaneously — messages flow in both directions at the same time.
rpc LiveChat (stream MessageRequest) returns (stream MessageResponse)
REST bridge: SSE (GET /api/chat/live-stream) + POST /api/chat/live-send

Disconnected — click Connect to open the live stream.

Open this tab in two browser windows simultaneously to see bidirectional, real-time communication between multiple clients.
